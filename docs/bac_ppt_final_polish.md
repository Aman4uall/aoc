# BAC PPT Final Polish and Assembly Guide

This file is the Phase 9 close-out guide for turning the BAC PPT draft into a clean final presentation.

Use it during actual slide assembly.

## 1. Final Build Order

Assemble the deck in this order:

1. Title and roadmap
2. Product / market / route basis
3. BFD and PFD slides
4. Reactor and equipment slides
5. Control and HAZOP slides
6. SHE / waste / site / layout slides
7. CAPEX / OPEX / working capital / finance slides
8. Conclusion

Reason:
- this order keeps the technical narrative stable before the economics closeout

## 2. Formatting Rules

### Titles

- keep slide titles short and direct
- use consistent capitalization across all slides
- do not let titles become paragraph-like

Recommended style:
- `Product Profile`
- `Reactor Design Basis`
- `Cost of Production`

### Body Text

- keep bullets short
- use sentence fragments only when clarity is not affected
- avoid walls of text
- limit most slides to 3-5 bullets when a figure is present

### Tables

- one visible main table per slide unless the slide is intentionally hybrid
- use consistent units in column headers
- use `Cr INR` for finance slides
- use `m3`, `kg/h`, `kW`, `bar`, `degC` consistently
- keep numeric precision limited to the agreed rounding rules

### Captions

- use one caption style throughout
- captions should explain the figure purpose, not restate the title

## 3. Slide Hygiene Rules

- every slide should have one main message
- remove any sentence that merely repeats the title
- if a figure is dominant, keep text secondary
- if a table is dominant, do not add a second large data block
- do not mix a long paragraph with a large table on the same slide

## 5. Table Polish Rules

- remove any row that does not change the slide’s conclusion
- if a table row is too wordy, shorten the label rather than shrinking the font
- for Slide 33 and Slide 35, visually separate the two data blocks so they do not feel like one oversized table
- do not copy any appendix-scale table into the main deck

## 6. Wording Rules To Preserve

Always preserve:
- `50 wt% sold-solution basis`
- `feasibility-grade`
- `screening-level`
- `model-based financial indicators`
- `glass-lined agitated vertical reactor with external recirculation heat-removal loop`

Avoid reintroducing:
- `Jacketed CSTR Train`
- `bankable`
- over-specific carrier language
- overly strong guarantee-style claims

## 7. Finance Slide Polish Rules

For Slide 35:

- highlight only 3-4 headline metrics visually:
  - annual revenue
  - annual operating cost
  - payback
  - IRR or break-even
- keep the caveat sentence visible
- do not make `0.48 y`, `60%`, and `4366.57 Cr INR` all the largest visual elements simultaneously

Recommended emphasis:
- one profitability metric
- one risk / qualification note
- one working-capital takeaway

## 8. Layout and SHE Slide Polish Rules

### Slide 29

- use a simple custom visual if possible
- if the visual is not ready, limit to three blocks:
  - safety
  - wastewater / CETP
  - hazardous waste handling

### Slide 31

- prefer zoning graphic over dense bullets
- keep the layout conceptual, not detailed-civil

### Slide 32

- if the utility routing graphic is not ready, use 4 compact bullets only

## 9. Final Pre-Submission Check

Before considering the PPT complete, verify:

- title slide matches the locked scope
- capacity is consistently `50,000 TPA`
- site is consistently `Dahej, Gujarat`
- product basis is consistently `50 wt% sold-solution basis`
- reactor wording is consistent everywhere
- no raw artifact-only wording appears in slides
- finance caveat remains visible
- all tables use consistent units
- all figures are readable at presentation size
- no slide feels crowded or duplicated

- the four existing BAC process/control graphics
- any 2-3 simple custom schematics made for layout / SHE / utility routing
- optional speaker notes based on the slide draft files
